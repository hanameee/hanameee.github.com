## 🙄 무슨 일...하십니까?

SW Frontend 인턴이다. 주로 로봇을 설정, 관리하고, 로봇의 데이터를 열람하기 위해 쓰이는 대시보드를 만드는 역할을 맡을 것 같다.

기술 스택은 Typescript + React + Material UI 를 사용한다. 협업은 Jira와 Gitlab으로.

1주일 간 일해본 결과, 아직 많은 부분이 jsx로 작성되어 있고 테스트코드가 없는 부분이 많아서 새로운 피쳐보다는 기존 기능의 버그를 잡고, tsx로 변환하고, 테스트코드를 작성하는 일을 주로 하지 않을까 생각하고 있다.

## 🔮 이번 주에 배운 것

### Git Git Git

- Jira + Gitlab 사용법을 배웠다.
- 1개의 일 = 1개의 Merge Request = 1개의 Branch = 1개의 Jira = 1개의 Commit
- 작업하면서 Remote Branch와 동기화 시키는 부분을 아직 잘 모르겠다. Rebase 공부가 필요해.
- 여러 작업을 해도 1개의 Commit 으로 Squash 해서 올려야 하기 때문에, 하나의 Commit을 템플릿에 맞게 명확하게 작성해야 한다. 다른 사람이 보기 쉽게 무엇을, 왜, 어떻게 수정했는지 기록한다.

### Best Practice

- 회사는 개인 프로젝트와 다르다. 서로 다른 여러 사람이 작성하는 것이기에 규칙이 있다. 그 규칙을 강제하기 위해 ESLint와 Prettier 과 같은 Formatter이 존재하고, 무조건 해당 규칙을 패스하도록 코드를 작성해야 한다.
- 디렉토리 구조, 테스트 방식, 함수 선언 위치, 타입 작성 등 모든 것에 Best Practice가 존재한다. 팀이 따르고 있는 방식이 Best Practice가 아닐 수 있으니, 항상 구글링하면서 공부할 것.

### Performance

- 잘 짜여진 프론트엔드의 결과는 속도가 말해준다. 항상 퍼포먼스에 주의하면서 코드를 작성해야 한다.
- 특히 Chrome 개발자도구에서 Performance 측정 (프로파일링) 하는 것을 잘 알아두어야 한다. Jest를 사용한 unit test로 퍼포먼스 측정을 하는 방법도 있지만, 테스트 코드에서 성능측정을 하는 것에 있어서는 찬반이 있는 모양.
